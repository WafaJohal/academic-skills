# Human Ethics Document Templates

These are the standard templates for the Plain Language Statement (PLS) and Consent Form.
When drafting these documents, populate every bracketed field from the interview data.
Omit [IF APPLICABLE] sections where not relevant. Retain all mandatory boilerplate exactly
as specified by your institution.

---

## DRAFTING GUIDANCE: When Multiple PLS/Consent Forms Are Needed

Different participant groups often need separate documents:

| Situation | Documents needed |
|-----------|-----------------|
| Single participant group, online survey | One PLS + one CF (or implied consent — CF not required if survey-proceed = consent) |
| Multiple phases with different activities | One PLS + one CF per phase (e.g. PLS_Phase1_Interviews, PLS_Phase2_Workshop) |
| Expert group (Delphi) + general participants | Separate PLS/CF for each group |
| Children | Child-appropriate PLS + parent/guardian PLS + parent CF + child assent form |
| Mixed online + in-person | Can share a PLS if activities described clearly; separate CFs for different consent modes |

When implied consent is used (online surveys where proceeding = consent):
- No separate consent form required — the PLS IS the consent document
- The PLS must include a statement such as: "By proceeding to the survey, you are indicating
  your consent to participate in this research."
- The application should still list a "Consent Form" attachment — upload the PLS in this slot,
  or note in the application that consent is implied

## VERSION CONTROL RULES

- Every revision creates a new version number (v1, v2, Delphi_v1, Delphi_v2, etc.)
- Version date and number must appear in the document footer
- When resubmitting after review, highlight changes in yellow or use tracked changes
- Remove superseded versions from Infonetica when uploading new versions
- The ethics ID only appears once approved — use placeholder [insert] until then
